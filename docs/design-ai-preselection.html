Scanner / GoldDigger AI 预选设计

状态：已实现。适用范围：Scanner 全市场扫股、沙里淘金 GoldDigger。

目标

在深度 AI 分析前增加轻量 AI 预选，让 LLM 参与“第一批进入深度分析的几十只股票”的选择，而不是只在规则漏斗完成后才做逐只研报分析。

该设计保留原有确定性筛选作为前置降噪层：行情、流动性、技术面、基本面、主题匹配和市场/行业分散仍由规则层完成；AI 预选只在较小候选池中做语义排序和取舍。

数据流

Scanner：股票池 → 元数据过滤 → 技术筛选 → 基本面筛选 → A股政策权重 → 行业/市场分散扩池 → AI 预选 → Tier 5 深度分析 → Top Picks。
GoldDigger：美股小盘/A股低位候选 → 超跌/估值/低覆盖过滤 → 主题检测 → 主题匹配评分 → 每市场扩池 → AI 预选 → 深度分析 → Gold Picks。

模块边界

模块职责输入输出
src/services/ai_preselector.py构造轻量 prompt、解析 JSON 排名、按 ticker 回填、失败回退已筛选候选、目标数量、Analyzer、可选模型进入深度分析的候选列表
MarketScanner把 Tier 4 从“直接截断”改为“扩池 + AI 预选”，并继续执行市场均衡护栏FundScore / CandidateStockTier 5 候选
GoldDigger按市场分别扩池并预选，避免美股/A股互相挤占每市场深度分析名额GoldCandidate每市场 Tier 5 候选

算法

Scanner

  1. Tier 3 后先按现有行业分散和跨市场保留逻辑生成预选池。
  2. 预选池大小为 min(Tier3 数量, SCANNER_MAX_TIER5_STOCKS * 2)，但不小于 SCANNER_MAX_TIER5_STOCKS。
  3. AI 根据候选的市场、行业、技术评分、趋势信号、基本面评分、估值、增长等字段，返回 JSON 排名。
  4. 系统只接受候选池内已有 ticker；缺失名额按原规则排序回填。
  5. AI 预选结果再经过跨市场保留，确保仍有有效候选的市场不会被单一市场完全挤出。

GoldDigger

  1. 主题匹配候选先按综合分排序。
  2. 每个市场取 max_tier5_per_market * 2 作为预选池。
  3. AI 根据超跌幅度、估值折价、主题相关性、机构信号和反转迹象，选择进入深度分析的 max_tier5_per_market 只。
  4. 每个市场独立预选，避免美股候选挤占 A股候选，或反过来。

Prompt 与解析

预选 prompt 要求 LLM 只返回 JSON 数组或包含 selected 字段的 JSON 对象，元素至少包含 ticker。系统兼容 ticker、symbol、code 三种字段名。

[
  {"ticker": "AAPL", "score": 92, "reason": "Quality setup and durable catalyst"},
  {"ticker": "300750", "score": 89, "reason": "Policy theme fit and improving setup"}
]

解析策略是严格白名单：返回 ticker 必须存在于输入候选池；重复、未知或空 ticker 会被忽略。

配置项

环境变量默认值语义
SCANNER_AI_PRESELECT_ENABLEDtrue是否启用 Scanner 深度分析前的 AI 预选。
GOLD_DIGGER_AI_PRESELECT_ENABLEDtrue是否启用沙里淘金深度分析前的 AI 预选。
SCANNER_MODEL空，继承全局模型Scanner AI 预选与 Tier 5 深度分析复用该模型。
GOLD_DIGGER_MODEL空，继承全局模型GoldDigger AI 预选与深度分析复用该模型。

失败与降级

  • LLM 调用异常：记录 warning，按原规则排序截断。
  • LLM 调用超过 120 秒：记录 warning，按原规则排序截断，避免 Scanner 长时间停在预选阶段。
  • JSON 解析失败：记录 warning，按原规则排序截断。
  • AI 返回未知 ticker：忽略未知 ticker，并用原规则排序补齐。
  • 候选数不足：直接返回现有候选，不额外调用 LLM。

边界条件

  • AI 预选不会扩大最终深度分析数量，只改变候选选择顺序。
  • Scanner 的最终 Top Picks 仍按深度分析置信度与规则综合分排序，并保留跨市场覆盖。
  • GoldDigger 仍先用新闻主题检测生成宏观主题，AI 预选只选择进入完整分析的股票。
  • A 股候选进入 AI 预选前会过滤指数、基金和 ETF；技术面/价格筛选通过统一数据源管理器 fallback，并对批量任务采用 Baostock 优先顺序，避免单一慢速行情接口失败导致 A 股候选全部丢失或长时间停滞。
  • 关闭预选开关后，行为回到规则排序 + 深度分析的旧路径。
