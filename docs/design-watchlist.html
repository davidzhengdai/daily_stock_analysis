自选股（Watchlist）功能设计文档

版本 1.0 · 2026-05-15 · 适用范围：stock-server + dsa-web

1. 背景与目标

系统原有的 watched_stocks 表属于 Sentinel 爬虫服务的内部状态， 不对用户直接暴露。用户需要一个独立的、面向业务的自选股列表，用于：

  • 集中管理长期关注的股票
  • 一键触发单只或批量分析任务
  • 在分析历史中通过"收藏"快速加入自选
  • 驱动 Sentinel 定向抓取（自动同步）

3. 数据模型

3.1 Watchlist（主库 stock_analysis.db）

字段类型说明
codeString(20) PK股票代码（大写规范化，如 AAPL、600519）
nameString(100)股票名称（可选）
added_atDateTime加入时间（UTC）
notesString(500)用户备注（可选）
last_analyzed_atDateTime nullable最近一次分析触发时间
规范化：所有写入路径在进入 WatchlistRepo 前将 code 转换为 .strip().upper()，以防大小写不一致导致重复记录。

5. REST API

方法路径描述
GET/api/v1/watchlist返回全部自选股，按 added_at 降序
POST/api/v1/watchlist添加/更新一只股票 {code, name?, notes?}，同步 Sentinel
DELETE/api/v1/watchlist/{code}移除股票，同步 Sentinel
PATCH/api/v1/watchlist/{code}更新 name/notes
POST/api/v1/watchlist/analyze批量触发分析 {codes?: list}，空 = 全部。返回 {submitted, codes}

6. Sentinel 同步机制

每次自选股写操作（add / remove）结束后，WatchlistService.sync_to_sentinel() 会将完整列表 upsert 到 Sentinel 的 watched_stocks 表。 Sentinel 每轮周期会对 watched_stocks 中的每只股票定向抓取 Google News RSS 及 StockAnalysis / Finviz HTML 新闻（美股）。

同步为本进程直接写 DB，Sentinel 需共享同一 SQLite 文件（Docker volume 挂载）。 若 Sentinel 未启用（SENTINEL_ENABLED=false），同步调用会被 try/except 静默跳过，不影响主流程。

7. Web UI 交互流

自选股页面（/watchlist）
  • 添加股票：输入框 + "添加" → POST /watchlist → 刷新列表 → 后台触发 Sentinel 即时抓取
  • 分析单只：行内"分析"按钮 → POST /watchlist/analyze 仅含该 code
  • 批量分析：勾选 checkbox → "分析选中 (N)" → POST /watchlist/analyze 含选中 codes
  • 分析全部：页头"分析全部" → POST /watchlist/analyze 无 codes（后端取全部）
  • 移除：行内删除按钮 → DELETE /watchlist/{code} → 刷新
分析历史 ⭐ 收藏（HomePage）
  • 页面加载时 GET /watchlist 取 watchedCodes: Set<string>
  • 每条历史记录右侧显示 Star 图标：已收藏填充金色，未收藏空心灰色
  • 点击 → 切换（POST or DELETE /watchlist/{code}）→ 乐观更新本地集合

8. 边界条件与注意事项

  • code 主键大小写敏感（SQLite 默认）— 写入路径统一 .upper() 规范化。
  • 分析任务通过任务队列异步提交（202），自选股页面仅展示"已提交 N 个任务"，不等待结果。
  • last_analyzed_at 当前由后端在 /analyze 端点提交时更新； 分析完成事件尚无回调写入，可作后续优化。
  • 自选股列表与持仓（Portfolio）相互独立，用途不同，不合并。
