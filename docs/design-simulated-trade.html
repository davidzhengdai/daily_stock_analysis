1. 功能概述（Feature Summary）

模拟交易（Simulated Stock Trade）是 DSA 系统的纸上交易练习模块，允许用户在不动用真实资金的前提下， 使用真实市场数据练习交易策略。系统在一个模拟账户下管理虚拟资金、持仓、订单与 AI 信号，全程与真实 行情数据打通，同时不与任何真实券商对接。

核心能力
  • ① 资金管理：支持 CNY / USD 双币种入金与出金，有资金台账记录
  • ② 多市场：A 股（CN，CNY）和美股（US，USD）
  • ③ 手动交易：支持限价单（Limit）和市价单（Market）的买入 / 卖出
  • ④ 自动交易：AI 驱动，基于自选股列表（Watchlist）定期触发信号与下单
  • ⑤ 互斥保护：自动交易开启后，手动交易入口禁用
  • ⑥ 空自选股守卫：自选股为空时不得开启自动交易，前端给出引导提示
  • ⑦ AI 增强：信号生成、仓位建议、止损/止盈、策略顾问、绩效归因
设计约束 本模块为单账户单实例设计（每个 DSA 部署一个模拟账户），无多用户隔离，无真实资金对接，无卖空（Short Selling）支持。

5. AI 信号生成算法（Signal Generation）

入口方法：SignalService.generate_signal(code: str, market: str) → SimulatedAISignal。 分为四个阶段：数据采集 → 技术预评分 → LLM 推理 → 风险覆盖层。

① 数据采集 → ② 技术预评分 → ③ LLM 推理 → ④ 风险覆盖 → ⑤ 写入 DB

5.1 数据采集

  • 从 data_provider 获取近 20 个交易日的 StockDaily（price、MA5/10/20、volume_ratio、pct_chg）
  • 从 simtrade_repo 读取当前持仓（qty、avg_cost、unrealized_pnl_pct）
  • 从 simtrade_repo 读取账户权益、可用现金、max_position_pct
  • 从 Sentinel 缓存读取最近新闻情绪摘要（可选，try/except 降级为 "N/A"）

5.2 技术预评分（rule-based，LLM 调用前）

在发起 LLM 请求前先用规则生成一个 technical_score（0.0–1.0）并注入 prompt， 帮助 LLM 快速定位技术形态，同时保留 LLM 对最终分值的调整权。

规则分值影响
MA5 > MA10 > MA20（多头排列）+0.25
MA5 < MA10 < MA20（空头排列）−0.25
当前价 > MA20（趋势偏多）+0.15
当前价 < MA20（趋势偏空）−0.15
volume_ratio > 1.5（放量）+0.10（方向由多空排列决定）
volume_ratio < 0.5（缩量）−0.05
近 5 日 pct_chg 累计 > +5%+0.10（动量延续）
近 5 日 pct_chg 累计 < −5%−0.10（动量下行）

初始基准值 0.5，加减后 clamp 到 [0.0, 1.0]。

5.3 LLM Prompt 结构

You are an AI trading advisor for a paper-trading simulation system.
Analyze the following data and generate a precise trading signal.

Stock: {code} ({name}), Market: {market}
Current Price: {price} {currency}
Technical Data (last 20 days):
  MA5={ma5}, MA10={ma10}, MA20={ma20}
  Volume Ratio={volume_ratio}
  5-day return={change_5d}%
  Pre-score: technical={technical_score:.2f}

Current Position: {qty} shares @ avg cost {avg_cost} (P&L: {pnl_pct}%)
Portfolio: equity={equity} {base_currency}, available cash={cash},
           max_position_pct={max_pos}%
News Sentiment: {sentiment_summary}
Auto-trade mode: {mode}
  (conservative=low risk, balanced=moderate, aggressive=growth)

Respond with JSON only:
{
  "signal": "buy|sell|hold|skip",
  "confidence": 0.0-1.0,
  "technical_score": 0.0-1.0,
  "sentiment_score": 0.0-1.0,
  "risk_score": 0.0-1.0,
  "position_size_pct": 0.0-100.0,
  "suggested_price": null or float,
  "stop_loss": null or float,
  "take_profit": null or float,
  "reasoning": "one concise sentence max 120 chars"
}

Guidelines:
- conservative: trade only on high conviction (confidence>0.75), small size (≤10%)
- balanced:     confidence>0.65, size ≤20%
- aggressive:   confidence>0.55, size ≤30%
- sell when:    stop-loss hit, take-profit hit, or signal reversal
- skip when:    data insufficient or position already at max

5.4 响应解析与校验

  1. 优先用 json.loads() 直接解析；失败则用 re.search(r'\{.*?\}', resp, re.DOTALL) 提取
  2. 所有数值字段 clamp 到合法范围；缺失字段填默认值
  3. 若 confidence < min_signal_confidence，强制将 signal 降为 "skip"
  4. LLM 调用失败（超时、5xx）：捕获异常，返回 signal="skip"、reasoning="LLM unavailable"，不抛出

5.5 风险覆盖层（Risk Management Overlay）

条件动作
该股持仓占比 ≥ max_position_pct 强制 signal → skip，rejection_reason = "position cap reached"
账户 max_drawdown_pct 已超限（status=paused） buy/hold → skip，仅允许 sell 通过
signal = sell 但当前 qty = 0 强制 → skip（无仓可卖）
计算所得 suggested_qty < 最小手数（CN:100，US:1） signal → skip，rejection_reason = "qty below min lot"
设计原则 风险覆盖层在服务层执行，不依赖 LLM 输出的准确性。LLM 负责策略，规则层负责风控， 两者各司其职。

6. 自动交易调度器（Auto-Trade Scheduler）

AutoTradeService 在服务启动时（server.py lifespan 事件） 启动一个 threading.Thread(daemon=True)，按配置间隔循环执行交易周期。

调度循环伪代码（每个 tick） 
interval = SIMTRADE_AUTO_TRADE_INTERVAL_MINUTES * 60  # 默认 5 min

while True:
    sleep(interval)

    account = repo.get_account()
    if not account.auto_trade_enabled or account.status != 'active':
        continue  # 自动交易未开启或账户已暂停，跳过

    watchlist = watchlist_repo.list_all()
    if len(watchlist) == 0:
        logger.warning("[SimTrade] Watchlist is empty, skipping auto-trade cycle")
        continue

    last_snapshot = repo.get_latest_snapshot()
    if last_snapshot and last_snapshot.max_drawdown_pct >= account.max_drawdown_pct:
        logger.warning("[SimTrade] Max drawdown exceeded, pausing buys")
        account_service.set_status('paused')
        continue

    # 批量过期旧信号
    repo.expire_old_signals()

    for stock in watchlist:
        signal = signal_service.generate_signal(stock.code, stock.market)
        repo.save_signal(signal)

        if signal.signal in ('buy', 'sell') and signal.confidence >= account.min_signal_confidence:
            order = order_service.place_order(
                code=stock.code,
                market=stock.market,
                side=signal.signal,
                order_type='limit' if signal.suggested_price else 'market',
                qty=signal.suggested_qty,
                limit_price=signal.suggested_price,
                source='auto',
                ai_signal_id=signal.id,
            )
            if order.status == 'filled':
                signal.status = 'executed'
                signal.order_id = order.id
                repo.update_signal(signal)

    # 巡检所有持仓的止损/止盈
    order_service.check_stop_loss_take_profit()

    # 每日快照（幂等，同一天只写一次）
    auto_trade_service.take_daily_snapshot()

    # 策略顾问（满 10 笔成交后生成绩效报告）
    if repo.count_filled_orders() % 10 == 0:
        auto_trade_service.generate_strategy_report()

6.1 止损 / 止盈巡检

order_service.check_stop_loss_take_profit() 遍历所有持仓， 获取当前最新价，判断是否触及 stop_loss_price 或 take_profit_price， 若触发则以市价单（source='auto'）平仓，并在 rejection_reason 填入触发原因。

价格刷新策略 巡检调用 data_provider.get_realtime_price(code)，若市场已收盘则使用昨日收盘价。 非交易时段触发的止损/止盈订单记录到 DB 但标记 status='pending'， 等待下一个交易时段的价格确认后再撮合。

6.2 每日快照计算

def take_daily_snapshot():
    today = date.today().isoformat()
    if repo.snapshot_exists(account.id, today):
        return  # 幂等，已存在则跳过

    positions = repo.list_positions(account.id)
    fx = float(os.getenv("SIMTRADE_USD_CNY_RATE", "7.25"))

    market_value_cny = sum(
        p.qty * p.last_price * (fx if p.currency == "USD" else 1.0)
        for p in positions
    )
    total_equity_cny = account.cash_cny + account.cash_usd * fx + market_value_cny

    total_deposited_equiv = account.total_deposited_cny + account.total_deposited_usd * fx
    total_return_pct = (total_equity_cny / total_deposited_equiv - 1) * 100 \
                       if total_deposited_equiv > 0 else 0.0

    prev_peak = repo.get_peak_equity(account.id) or total_equity_cny
    peak_equity = max(prev_peak, total_equity_cny)
    max_drawdown_pct = (peak_equity - total_equity_cny) / peak_equity * 100 \
                       if peak_equity > 0 else 0.0

    repo.upsert_snapshot(SimulatedSnapshot(
        account_id=account.id, date=today,
        cash_cny=account.cash_cny, cash_usd=account.cash_usd,
        market_value_cny=market_value_cny, total_equity_cny=total_equity_cny,
        realized_pnl=sum(p.realized_pnl for p in positions),
        unrealized_pnl=sum(p.unrealized_pnl for p in positions),
        total_return_pct=total_return_pct,
        max_drawdown_pct=max_drawdown_pct, peak_equity_cny=peak_equity,
    ))

7. 佣金与撮合规则（Commission & Fill Rules）

7.1 佣金计算

市场方向规则最低
CN（A 股）买入成交金额 × 0.025%5 CNY
CN（A 股）卖出成交金额 × 0.025% + 成交金额 × 0.1%（印花税）5 CNY（佣金部分）
US（美股）买入max($1.00, $0.005 × shares)$1.00
US（美股）卖出max($1.00, $0.005 × shares)$1.00
实现位置：order_service._calc_commission(side, market, qty, fill_price, currency)， 返回佣金金额（与订单同币种）。

7.2 订单撮合规则

订单类型撮合规则备注
市价单（Market） 立即成交，成交价 = last_price × slippage
买入：slippage = 1.001（+0.1%）
卖出：slippage = 0.999（−0.1%） 		模拟市场冲击
限价单（Limit） 若 |limit_price − last_price| / last_price ≤ 3% → 立即以 limit_price 成交
否则标记 pending，等待价格刷新 		3% 偏差阈值可在 config 中调整
Pending 限价单 调度器每轮或价格刷新时重新检查；超过 1 个交易日（T+1 收盘）未成交则自动撤销 退还冻结资金

7.3 最小交易单位

市场最小手数说明
CN（A 股）100 股 / 手qty 必须是 100 的倍数
US（美股）1 股支持零股（fractional shares 不支持，整数股）

8. 汇率处理（FX Rate）

USD/CNY 汇率通过环境变量 SIMTRADE_USD_CNY_RATE（默认 7.25）静态配置， 不调用实时汇率 API（避免额外依赖，也符合模拟场景的简化需求）。

汇率使用场景：

  • 每日快照：USD 持仓和现金折算为 CNY 计算 total_equity_cny
  • 仓位上限检查：跨币种组合时，将 USD 仓位折 CNY 后与总权益对比
  • 前端账户概览：统一以基准货币（默认 CNY）展示总资产
注意：汇率仅用于展示和仓位计算，不用于订单资金扣除， 订单冻结和扣款始终使用订单本身的货币（CNY 订单扣 CNY，USD 订单扣 USD）。

10. AI 增强能力清单（AI-Enhanced Features）

#能力实现位置说明
1 信号生成 AI signal_service.generate_signal() LLM 综合技术面 + 情绪面输出结构化交易建议（buy/sell/hold/skip）
2 自适应仓位 AI signal_service → order_service LLM 输出的 position_size_pct 随置信度和账户模式变化，服务层换算为实际手数
3 动态止损/止盈 AI signal_service → SimulatedPosition LLM 根据 MA 支撑/压力位建议绝对价格，写入持仓的 stop_loss_price / take_profit_price
4 市场趋势感知 AI Prompt context Prompt 注入 MA 多空排列描述（bull/bear/sideways），引导 LLM 把握宏观方向
5 策略顾问报告 AI auto_trade_service.generate_strategy_report() 每满 10 笔成交，用 LLM 生成绩效摘要（胜率、AI 信号 vs 手动对比），写入日志/信号表
6 交易日记 SimulatedAISignal.reasoning 每笔自动交易保存 AI reasoning，信号历史表可读，形成可追溯的决策链
7 回撤保护 auto_trade_service 自动监控快照中 max_drawdown_pct，超限后自动暂停所有买入，账户切换为 paused
8 绩效归因 SimulatedAISignal + SimulatedOrder 信号历史通过 order_id 关联订单，可追溯哪些信号带来盈利/亏损

12. 非目标 / 范围边界（Non-Goals）

不在范围内说明
真实资金 / 券商接入本模块仅为模拟演练，不与任何真实交易系统对接
订单簿模拟（买卖盘撮合）不模拟 bid/ask spread，直接使用最新价加滑点
历史回测（Backtesting）回测为独立功能，数据要求和计算架构不同，不在本模块范围
多用户隔离每个 DSA 实例一个模拟账户，无用户认证隔离
卖空（Short Selling）不支持，sell 订单只能基于现有持仓
期权、ETF 杠杆、衍生品仅支持普通股票买卖
实时汇率汇率静态配置，不调用实时 API
分红 / 拆股处理不自动调整持仓的分红和拆股
